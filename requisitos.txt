Microsserviço de Logística de Entrega:

Descrição Funcional: este serviço cuidará de toda a logística
relacionada à entrega de pedidos, desde a atribuição de entregadores até o
rastreamento das entregas em tempo real. A função inclui calcular as rotas mais
eficientes, estimar tempos de entrega e fornecer atualizações de status aos
clientes.


Rotas de Entrega: o sistema deve ter 1 endpoint que receba um cep (ou parte dele) e agrupe pedidos pendentes de entrega pelo CEP recebido.
    Retornando esses pedidos (ids) como resposta.
    8.1 - Infos sobre CEP: https://www.correios.com.br/enviar/precisa-de-ajuda/tudo-sobre-cep

    8.2 - Opcional: Se quiserem fazer uma solução mais robusta integrando com google maps ou outro equivalente, fiquem a vontade.
        Documentação: façam uma boa documentação pois ela será o guia na avaliação. Sugiro que façam um vídeo (de 10 min) apresentando os pontos implementados,
        debitos técnicos, etc
        Avaliação: O avaliador além "buildar" e "rodar" a aplicação, irá certificar todo código implementado (como se fosse um code-review).
    10.1 - Então o mais importante não são as regras perfeitamente codadas (aceitamos alguns bugs e debitos técnicos),
        mas sim a boa pratica e correto uso dos frameworks.
        Todos os requsitos aqui citados ou existente no TC serão repassados nas Lives e nos Grupos de Estudos.
         Além disso estarei disponivel no Discord praticamente todos os dias. Se tiverem dúvidas ou travas me procure.

Rastreamento de Pedidos: O sistema deve ter 1 endpoint que receba a latitude e longitude (e mais alguns outros dados como id do pedido, etc)
        que irá armazenar essa info no database.
        7.1 - Opcional: Se quiserem fazer uma solução mais robusta integrando com google maps ou outro equivalente, fiquem a vontade.

Tecnologias e Abordagens:
•Spring Boot para a base do microsserviço.
•Spring Data JPA para persistência de dados.

- Considerar que os pedidos são recebidos já no centro de distribuição, e que serão despachados a partir de lá.

- verificar pedidos para serem processados
    - Receber via evento os pedidos com dados de entrega e gravar na base
        - Ao salvar na base colocar o pedido com status "Pedido Recebido"
    - Criar uma batch que busca pedidos na base que devem ser despachados no dia.
        - Ao busca os pedidos atualizar o status para "Em Preparação"

- despachar pedidos
    - verificar pedidos por cep/região
    - atribuir entregador
    - calcular rota
    - estimar tempo de entrega
    - atualizar status do pedido ( Em preparação, Em trânsito, Entregue )
    - enviar notificação para o cliente ( Via email logistica@mscode.com.br)

- rastrear entregas
    - Serviço para rastrear em tempo real a entrega de pedidos
    - Usar a API Rota do Google Directions API para calcular a rota e estimativa de tempo de entrega.
    - Para saber o tempo de entrega, passar cep inicial e final, e tempo de saida ( departure_time ).
